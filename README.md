## Business Problem
E-commerce platforms overwhelm customers with thousands of products.
A recommendation system personalises product discovery, driving higher purchases,
longer sessions, and repeat visits. Amazon attributes ~35% of revenue to recommendations.

## Dataset
- **Amazon Product Ratings** from Kaggle
- URL: https://www.kaggle.com/datasets/skillsmuggler/amazon-ratings
- File: `ratings_Beauty.csv`
- Rename to `ratings.csv` and place in `data/`
- Columns: userId, productId, rating, timestamp

## Setup & Run

### 1. Create virtual environment (Windows)
```bash
python -m venv venv
venv\Scripts\activate
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Run EDA notebook
```bash
jupyter notebook notebooks/eda.ipynb
```
This generates `data/ratings_filtered.csv`

### 4. Train the models
```bash
jupyter notebook notebooks/model_training.ipynb
```
Run all cells — this saves all model artifacts to `app/`

### 5. Launch Streamlit app
```bash
cd app
streamlit run streamlit_app.py
```

## Models Implemented
| Approach                | Method               | Metric       |
|-------------------------|----------------------|--------------|
| Content-Based Filtering | Cosine Similarity    | Similarity % |
| User-Based CF           | Cosine on UI Matrix  | RMSE, MAE    |
| Item-Based CF           | KNNWithMeans         | RMSE, MAE    |
| Matrix Factorization    | SVD (Surprise)       | RMSE, NDCG@K |
| Hybrid                  | SVD + Content (0.7/0.3) | Combined  |

## Evaluation Metrics
- **RMSE / MAE** — rating prediction error
- **Precision@K** — fraction of recommended items that are relevant
- **Recall@K** — fraction of relevant items that were recommended
- **NDCG@K** — ranking quality (rewards correct ordering of relevant items)

## Cold Start Strategy
- New users → Content-based fallback (recommend top-rated popular products)
- New products → Cannot be recommended until sufficient ratings accumulate
- Minimum thresholds: user >= 5 ratings, product >= 10 ratings
